# Техническая спецификация Git Client для Raycast

## Обзор проекта

Raycast расширение для управления локальными Git-репозиториями с интуитивным интерфейсом, предоставляющее функциональность управления ветками, файлами, коммитами и stash через нативный API Raycast.

### Архитектура расширения

Расширение построено на принципах:
- **Модульная архитектура** с разделением ответственности
- **Нативное использование Raycast API** без излишних абстракций
- **Переиспользование компонентов** для Action и UI состояний
- **Синхронная валидация** для быстрой обратной связи
- **Frecency-сортировка** для недавних репозиториев

## Реализованная функциональность

### ✅ Основные команды

1. **`open-repository`** - Основная команда для работы с Git репозиторием
   - Принимает путь к репозиторию через аргументы
   - Поддерживает пути с тильдой (`~`)
   - Синхронная валидация репозитория
   - Автоматическое добавление в недавние репозитории
   - Навигация между 4 представлениями (branches, status, commits, stashes)

2. **`recent-repositories`** - Команда для работы с недавними репозиториями
   - Frecency-сортировка (частота + недавность)
   - Автоматическое кеширование между сессиями
   - Действия: открыть, показать в Finder, копировать путь, создать QuickLink
   - Возможность очистки списка

### ✅ Представления (Views)

#### 1. **Branches View** - Управление ветками
- **Группировка:** Текущая ветка, локальные ветки, удаленные ветки (по remote)
- **Информация:** Статус ahead/behind, upstream информация
- **Действия для текущей ветки:**
  - Pull, Push, Fetch, Создать ветку
- **Действия для локальных веток:**
  - Checkout, Удалить (с подтверждением), Push, Pull, Merge, Rebase
- **Действия для удаленных веток:**
  - Checkout как локальная ветка, Удалить remote ветку (с подтверждением)

#### 2. **Status View** - Управление файлами
- **Секции:** Staged, Unstaged, Untracked файлы
- **Diff preview** в Detail View для каждого файла
- **Действия с файлами:**
  - Stage/Unstage, Discard changes (с подтверждением)
  - Открыть в редакторе, показать в Finder, копировать путь
  - Удалить untracked файлы
- **Глобальные действия:**
  - Stage/Unstage all, Commit с формой
  - Быстрое commit с последним сообщением

#### 3. **Commits View** - История коммитов
- **Фильтрация:** Dropdown для выбора ветки (All Branches + конкретные ветки)
- **Информация:** Сообщение, автор, дата, короткий хеш
- **Diff preview** для каждого коммита
- **Действия с коммитами:**
  - Cherry-pick, Revert, Reset (с подтверждением)
  - Копировать хеш, показать файлы
  - Создать тег

#### 4. **Stashes View** - Управление stash
- **Действия:** Apply, Pop, Drop stash (с подтверждением)
- **Создание stash:** Форма с опциональным сообщением
- **Oтображение:** Название, дата, сообщение stash

### ✅ Компоненты архитектуры

#### Hooks (Кастомные React хуки)
- **`useGitRepository`** - Синхронная валидация и создание GitManager
- **`useGitBranches`** - Загрузка и кеширование списка веток
- **`useGitStatus`** - Загрузка статуса файлов
- **`useGitCommits`** - Загрузка истории коммитов с пагинацией
- **`useGitStash`** - Управление stash
- **`useGitDiff`** - Получение diff для файлов и коммитов
- **`useRecentRepositories`** - Управление недавними репозиториями с frecency
- **`useCommitsBranchFilter`** - Фильтрация коммитов по веткам

#### Action Components (Переиспользуемые действия)
- **`BranchActions`** - Действия с ветками по типу
- **`FileActions`** - Действия с файлами по статусу
- **`CommitActions`** - Действия с коммитами и историей
- **`StashActions`** - Действия со stash

#### Shared Components (Общие UI компоненты)
- **`ErrorView`** - Отображение ошибок с возможностью повтора
- **`EmptyView`** - Пустые состояния с описанием и действиями
- **`LoadingView`** - Индикаторы загрузки

#### Utils (Утилиты)
- **`GitManager`** - Основной класс для всех Git операций
- **`validation`** - Валидация Git репозиториев и путей с тильдой

### ✅ Технические особенности

#### Git Operations
- **Библиотека:** simple-git для всех Git операций
- **Таймауты:** 10 секунд для Git операций
- **Производительность:** Ограничения на количество элементов (200 веток, 500 файлов)
- **Логирование:** Консольное логирование для отладки

#### UI/UX
- **Иконки:** Встроенные иконки Raycast (не эмодзи)
- **Цветовая схема:** Стандартная цветовая палитра Raycast
- **Сокращения клавиш:** Для всех основных действий
- **Подтверждения:** Для всех деструктивных операций

#### Error Handling
- **Паттерн:** Прямые вызовы `showToast()` без обёрток
- **Безопасность:** Проверка `error instanceof Error`
- **Fallback:** Для неизвестных ошибок
- **Валидация:** Синхронная валидация репозиториев

#### Performance
- **Кеширование:** `useCachedPromise` для всех операций загрузки
- **Frecency:** Встроенная frecency-сортировка для недавних репозиториев
- **Пагинация:** Ограничения на количество загружаемых элементов
- **Ленивая загрузка:** Detail View загружается по требованию

## Принципы разработки

### Архитектурные принципы
1. **Команды содержат весь UI код** - никаких отдельных компонентов для интерфейсов
2. **Переиспользуемые Action компоненты** - вынесены в `/components/actions/`
3. **Минимальные утилиты** - только необходимые файлы
4. **Нативное Raycast API** - без излишних абстракций

### Code Style
1. **Именование:**
   - Команды: `kebab-case`
   - Компоненты: `PascalCase`
   - Хуки: `camelCase` с префиксом `use`
   - Типы: `PascalCase`

2. **Error Handling:**
   - Прямые вызовы `showToast()` без обёрток
   - Проверка `error instanceof Error`
   - Подтверждения для деструктивных операций

3. **Performance:**
   - Кеширование через `useCachedPromise`
   - Ограничения на количество элементов
   - Frecency для недавних репозиториев

## Особенности реализации

### GitManager класс
- **Центральный компонент** для всех Git операций
- **Путь к репозиторию** доступен через публичное поле `repoPath`
- **Название репозитория** доступно через getter `repoName`
- **Консольное логирование** для отладки
- **Таймауты** для операций

### Валидация репозиториев
- **Синхронная валидация** для быстрой обратной связи
- **Поддержка тильды** в путях (`~`)
- **Проверка .git папки/файла**
- **Информативные сообщения об ошибках**

### Frecency для недавних репозиториев
- **Автоматическое кеширование** между сессиями
- **Обновление рейтинга** при использовании
- **Очистка списка** с подтверждением
